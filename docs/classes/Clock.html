Clock Class

Defined in: src\time\Clock.ts:10
Module: Time
Parent Module: Kiwi

The Clock class offers a way of tracking time within a game. When creating a new Clock you should NOT directly instantiate this class but instead use the addClock method on a ClockManager. - The MasterClock is a property of the Kiwi.Time.Manager class and tracks real world time in milliseconds elapsed since the application started. This happens automatically and there is no need to do anything to set this up. - An instance of a clock is used to track time in arbitrary units (milliseconds by default) - A clock can be started, paused, unpaused and stopped. Once stopped, re-starting the clock again will reset it. - Any number of timers can be attached to a clock. See the Kiwi.Time.Timer class for timer details. - If the clock is paused, any timers attached to the clock will take this into account and not continue to fire events until the clock is unpaused. (Note that this is not the same as pausing timers, which can be done manually and needs to be undone manually.)
